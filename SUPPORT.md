# Support

This project is maintained on a best-effort basis.

## Where to Ask For Help

- **Bug reports**: GitHub Issues (use the bug report template)
- **Feature proposals**: GitHub Issues (use the feature request template)
- **Usage questions**: GitHub Discussions or Issues
- **Security issues**: follow `SECURITY.md` (private reporting path first)

## What Maintainers Can Usually Help With

- Reproducible compiler/runtime/tooling defects
- Conformance or strict-mode expectation mismatches
- Documentation gaps affecting contributor onboarding
- Clarifying canonical vs compatibility behavior in current release surfaces

## What May Not Receive Immediate Support

- Custom deployment architecture consulting
- Priority feature delivery timelines
- Environment-specific debugging without a minimal reproduction
- Third-party integrations outside maintained adapters/tooling

## Support Boundaries

- AI Native Lang (AINL) has mixed maturity across subsystems; check `docs/CONFORMANCE.md` and `docs/RELEASE_READINESS.md` before assuming production stability for every surface.
- Example/corpus strictness expectations are defined in `tooling/artifact_profiles.json`.
- Compatibility behavior may be retained intentionally and is not always a strict conformance target.
